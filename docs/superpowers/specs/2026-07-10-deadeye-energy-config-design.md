# Deadeye 能量配置调整设计

## 目标

调整 Deadeye 的 Cloth Config 配置界面，并增加服务端权威的无限能量选项：

- 持续时间滑条仅提供整数 1～30 秒。
- 恢复延迟滑条提供 0～10 秒，步进为 0.25 秒。
- 增加默认关闭的无限能量开关。
- 能量为 100% 时始终隐藏能量数值；无限能量模式不显示能量数值。

## 配置模型

在 `deadeye-common.toml` 的 `[deadeye]` 节新增布尔配置：

```toml
infiniteEnergy = false
```

该配置属于 COMMON 配置，由服务端决定实际游戏行为。多人游戏中，客户端本地副本不能覆盖远程服务端设置。

现有 `energyDurationSeconds` 和 `energyRecoveryDelaySeconds` 的 Forge 配置范围及命令范围保持不变。本次只收窄 Cloth Config 界面的输入范围，避免破坏已有 TOML 文件和服务端命令兼容性。

## Cloth Config 界面

### 持续时间

继续使用整数滑条，范围由 1～60 调整为 1～30，默认值保持 5 秒。若已有配置值超出界面范围，打开界面时将显示为最近的合法边界值。

### 恢复延迟

Cloth Config 的整数滑条不直接支持 0.25 步进，因此界面内部使用 0～40 的整数刻度：

- 显示值：`刻度 / 4.0` 秒。
- 保存值：`刻度 / 4.0` 写回 `ENERGY_RECOVERY_DELAY_SECONDS`。
- 读取值：将秒数乘以 4、四舍五入，并限制到 0～40。

显示文本去掉无意义的尾随零，例如 `0s`、`0.25s`、`0.5s`、`0.75s`、`1s`。

### 无限能量

在“常规”分类中增加布尔开关，默认关闭。保存时写入 `DeadeyeConfig.INFINITE_ENERGY`。

## 服务端能量行为

无限能量关闭时维持现有消耗、耗尽强制释放、恢复延迟和恢复速度逻辑。

无限能量开启时：

1. `canActivate` 始终允许玩家激活死亡之眼。
2. 每个服务端 Tick 将在线玩家能量规范化为 100%。
3. 跳过能量消耗、耗尽释放、恢复延迟和恢复计算。
4. 当能量从非满值变为 100% 时，沿用现有能量同步数据包通知对应客户端。
5. 关闭无限能量后，玩家从 100% 开始恢复正常消耗。

配置在玩家按住死亡之眼期间切换时，无需重新按键；下一服务端 Tick 即采用新模式。

## 客户端 HUD 行为

能量文字只在能量严格低于满值阈值 `99.95%` 时渲染。达到 100% 后，无论死亡之眼是否激活，都不显示能量数值。

无限能量模式由服务端将能量同步为 100%，因此无需新增网络数据包或客户端无限模式状态。眼睛图标、渐晕和淡入淡出保持原有行为。

## 本地化和文档

为无限能量开关增加简体中文和英文名称、说明文本。README 的公共配置表增加 `infiniteEnergy`，并说明 Cloth Config 的持续时间及恢复延迟界面范围。

## 测试策略

项目当前没有测试文件。本次增加 JUnit 5 测试基础设施，并优先测试不依赖正在运行的 Minecraft 实例的纯逻辑：

- 恢复延迟秒数与四分之一秒滑条刻度之间的双向换算和边界限制。
- 无限能量开启时允许零能量玩家激活。
- 无限能量开启时能量被规范化为 100%，关闭时保留正常能量值。
- 普通模式下的能量扣减行为不被无限能量改动破坏。
- 满能量 HUD 可见性判断始终返回隐藏，低于满值时返回显示。

完成后运行目标测试、完整测试任务和 Gradle 构建。

## 不在本次范围内

- 不增加 `/deadeye infinite` 命令。
- 不改变现有 TOML 和命令对持续时间、恢复延迟的合法范围。
- 不持久化玩家能量。
- 不修改网络协议或增加新的同步数据包。
